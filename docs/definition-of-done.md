# Definition of Done

本文档定义每个功能的完成标准。未来 agent 完成任务前必须逐项检查。

## Universal Done

所有任务必须满足：

- 实现符合 `AGENTS.md` 和相关 `docs/` 文档。
- 没有扩大范围做无关功能。
- 没有引入无理由的新框架或大依赖。
- 代码符合项目现有风格。
- 关键逻辑有测试。
- 错误处理使用统一错误码。
- 日志包含必要上下文，但不泄漏敏感信息。
- 新增配置有默认值和文档。
- 数据库变更有 migration。
- 对外 API 变更同步 OpenAPI 或接口文档。
- 最终回复列出验证结果。

## Feature Done Checklist

### API

- 请求参数有验证。
- 返回格式统一。
- 错误码稳定。
- 鉴权和权限明确。
- 分页接口有 limit 上限。
- 幂等接口有幂等策略。
- OpenAPI 文档更新。

### Database

- 有 migration。
- 有必要索引。
- 外键、唯一约束或业务唯一约束明确。
- 软删除和硬删除策略明确。
- 时间字段使用统一时区策略。

### Redis

- Key 命名稳定。
- TTL 策略明确。
- 高风险写入有幂等或事务策略。
- 集成测试覆盖关键操作。

### gRPC and Protobuf

- Protobuf 字段编号稳定。
- 删除字段必须 reserved。
- gRPC 错误映射到统一业务错误码。
- 生成代码流程可重复执行。
- 公开 service 有契约测试或集成测试。

### TCP

- 连接时鉴权。
- 包头和 payload 长度校验完整。
- 未认证连接只能发送 auth、ping、pong。
- 断线处理明确。
- 心跳和超时策略明确。
- 消息类型稳定。
- 错误消息格式统一。
- 不能向无权限用户广播敏感消息。

### Security

- 密钥不进入源码。
- 用户输入经过验证。
- 后台接口有 RBAC。
- 高风险操作有审计。
- Token 可撤销。
- 平台回调有验签或来源校验。

### Tests

- 单元测试覆盖核心业务规则。
- API 集成测试覆盖主要成功/失败场景。
- 平台适配器有 contract tests。
- Redis 相关能力有集成测试。
- TCP 长连接能力有集成测试。
- gRPC service 有必要的契约测试或集成测试。

### Documentation

需要在以下情况更新文档：

- 新增模块。
- 改变模块边界。
- 新增公开 API。
- 新增数据库核心模型。
- 新增平台适配器。
- 改变排行榜、聊天、账号系统行为。
- 改变部署、配置或验证命令。

## MVP Done

MVP 版本完成时必须具备：

- 本地服务可启动。
- 用户可登录。
- Steam/Epic 至少一个真实或模拟验证链路可运行。
- 用户可提交排行榜分数。
- 用户可查询排行榜。
- 用户可建立 TCP 聊天连接。
- 用户可发送和接收聊天消息。
- 管理员可封禁用户。
- 被封禁用户不能登录或聊天。
- 有基础测试套件。
- 有本地开发说明。
