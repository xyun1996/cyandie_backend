# Module Boundaries

模块边界是保证 agent 不把系统写成一锅粥的关键约束。

## General Rules

- 模块只能通过公开 service/interface 调用其他模块。
- 禁止跨模块直接访问对方的数据库 repository，除非该 repository 属于 `shared` 基础设施。
- 跨模块业务动作优先使用应用服务编排。
- 共享类型放在 `shared` 或模块自己的 `contracts` 中。
- 不要把业务逻辑塞进 controller/gateway。
- 不要让第三方平台 SDK 泄漏到业务模块。
- 不要让 TCP 包解析、Protobuf 编解码细节泄漏到业务服务。

## Auth

职责：

- 登录。
- Token 签发和刷新。
- Session 管理。
- 账号绑定和解绑。
- 账号合并流程。
- 调用平台适配器验证身份。

可以调用：

- `users` 查询和创建本地用户。
- `platforms` 验证平台身份。
- `admin/audit` 写安全事件。

不允许：

- 写排行榜。
- 发送聊天消息。
- 修改背包或奖励。
- 直接调用 Steam/Epic SDK。

## Users

职责：

- 本地用户实体。
- 用户资料。
- 用户状态。
- 封禁状态。
- 用户查询。

可以被：

- Auth。
- Chat。
- Leaderboards。
- Admin。
- Friends。

不允许：

- 签发 Token。
- 验证平台 Token。
- 直接处理支付。

## Platforms

职责：

- 定义 `PlatformAdapter`。
- 维护平台适配器注册表。
- 调用平台 API。
- 标准化平台用户、购买、成就、统计数据。

不允许：

- 创建本地用户。
- 签发 JWT。
- 写入排行榜。
- 发送聊天消息。
- 写入后台审计，除非是平台调用日志。

## Leaderboards

职责：

- 榜单配置。
- 分数提交。
- 排名查询。
- 赛季。
- 快照。
- 奖励结算入口。

可以调用：

- `users` 查询用户基础状态。
- `admin/audit` 记录后台配置变更。
- `inventory` 发放赛季奖励，必须通过公开奖励接口。

不允许：

- 验证平台身份。
- 修改用户账号。
- 发送聊天消息。
- 直接写背包表。

## Chat

职责：

- TCP 长连接。
- TCP 包编解码。
- 连接鉴权、心跳和断线处理。
- 私聊。
- 频道聊天。
- 群聊。
- 消息存储。
- 离线消息。
- 禁言。
- 举报。
- 黑名单影响聊天。

可以调用：

- `users` 查询用户状态。
- `friends` 查询黑名单或好友关系。
- `admin/audit` 记录禁言等后台动作。

不允许：

- 修改用户身份。
- 写排行榜。
- 处理支付。
- 直接调用平台 SDK。

## Protocol

职责：

- Protobuf schema。
- gRPC service 定义。
- TCP 消息类型。
- 生成代码。
- 协议兼容规则。

不允许：

- 包含具体业务流程。
- 直接访问数据库。
- 直接调用第三方平台 SDK。

## Friends

职责：

- 好友申请。
- 好友关系。
- 黑名单。
- 最近联系人。
- 在线状态接口。

可以调用：

- `users` 查询用户存在性和状态。
- `platforms` 的平台好友导入能力，但必须通过适配器接口。

不允许：

- 发送聊天消息本体。
- 签发 Token。
- 写排行榜。

## Admin

职责：

- 后台 API。
- RBAC。
- 用户管理。
- 榜单配置。
- 聊天审核。
- 订单查询。
- 审计日志。

可以：

- 编排多个模块。

必须：

- 对高风险操作写审计日志。
- 校验后台角色权限。
- 记录操作者、操作对象、原因、前后状态。

不允许：

- 绕过业务模块直接改核心数据，除非是专门设计的 admin repository 且有审计。

## Payments

职责：

- 订单。
- 平台收据验证。
- Webhook。
- 发货流程。
- 幂等。

可以调用：

- `platforms` 验证购买。
- `inventory` 发货。
- `admin/audit` 写订单状态变化。

不允许：

- 直接签发用户登录 Token。
- 写聊天、好友、排行榜数据。

## Inventory

职责：

- 背包。
- 货币。
- 奖励发放。
- 资产流水。

必须：

- 保证发放幂等。
- 记录资产变化流水。

不允许：

- 自己决定平台购买是否有效。
- 直接处理平台 webhook。

## Shared

职责：

- 错误码。
- 日志。
- 配置。
- 基础类型。
- 验证工具。
- 时间、ID、分页等通用工具。

不允许：

- 依赖任何业务模块。
- 包含具体业务流程。
